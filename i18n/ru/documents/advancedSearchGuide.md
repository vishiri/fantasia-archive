# Руководство по расширенному поиску

---

## Введение

Fantasia Archive поставляется с довольно продвинутой поисковой системой в большинстве полей поиска, которая может выполнять поиск как по всем типам документов, так и по определенным типам, например, по множественным и одиночным полям связи на каждой странице документа и всплывающему окну быстрого поиска.

---

## Интеллектуальное поисковое сопоставление и сортировка

Сам поиск работает следующим образом: вы можете ввести любое количество слов, и программа будет обрабатывать их индивидуально, если они разделены пробелом.

### Поиск осуществляется по этим правилам

- **Поиск осуществляется без учета регистра, что означает, что вы можете вводить все в ВЕРХНЕМ или нижнем регистре (или любым другим способом), это не имеет значения**
- **Слова могут быть в любом порядке**
  - Пример: «Темный страшный замок» будет найден, даже если вы наберете «Темный страшный замок».
- **Даже части слов могут дать удачное совпадение**
  - Пример: «Темный страшный замок» будет найден, даже если вы наберете «sca tle ark».
- **Документы сортируются по приоритету по следующим правилам:**
  1. **Прямое совпадение имеет приоритет над всем остальным**
      – Пример: «Темный страшный замок» напрямую соответствует запросу, содержащему слово «темный страшный замок».
  2. **Полное совпадение слов имеет приоритет над фрагментами**
      - Каждое полностью совпавшее слово учитывается индивидуально; чем больше полных совпадений имеет документ, тем выше он будет в списке
      – Пример: слово «Dark Scary Castle» имеет два полных совпадения со словом «dark Scary tle».
  3. **Фрагменты находятся внизу списка**
      - Каждый совпавший фрагмент засчитывается индивидуально; чем больше фрагментов имеет документ, тем выше он будет в списке
      - Пример: «Темный страшный замок» имеет 2 совпадения фрагмента из «Sc tle».- **Вы можете включить в поиск `Другие имена`, добавив к строке поиска префикс `@`**
  - Пример: `@Логово вампира` (если в вашем документе `Темный страшный замок` в разделе Другие имена указано `Логово вампира`, поиск найдет его таким образом)

## Фильтрация

В дополнение к расширенным функциям поиска Fantasia Archive также предлагает мгновенную фильтрацию по нескольким атрибутам для дальнейшего сужения результатов поиска.

- **ПРИМЕЧАНИЕ. Все следующие значения фильтров (включая фильтрацию полного поиска в следующем разделе) поддерживают сопоставление любой части искомого текста с любой частью поискового запроса**
  – Пример: `>nada` будет соответствовать `Continent > North America > Canada > Toronto`.

### Фильтрация работает следующим образом и следует следующим правилам:

- **Любое из следующих условий фильтра НЕ БУДЕТ конфликтовать с обычным поиском слов; поэтому вы можете использовать их вместе**
- **Вы можете использовать только один экземпляр каждого из следующих типов фильтров одновременно; однако разные типы фильтров могут быть активны вместе**
- **Фильтр не чувствителен к регистру, что означает, что вы можете вводить все в ВЕРХНЕМ или нижнем регистре, это не имеет значения**
- **Если термин фильтра содержит пробелы, замените их символом `-`**
  - Пример: вы хотите выполнить поиск по тегу «Персонажи игроков». Чтобы полностью соответствовать этому тегу, вам нужно будет ввести «#player-characters».
- **Фильтр иерархического пути автоматически удаляет из пути все символы `>`; это приводит к их исключению из строки фильтра**
  - Пример: вы хотите найти иерархический путь, содержащий следующее `США > Вирджиния > Ричмонд`. Чтобы полностью соответствовать этому иерархическому пути, вам нужно будет ввести `>usa-virginia-richmond`
- **Можно использовать следующие условия фильтрации**
  - `$` - Символ для поиска типа документа.
  - `#` - Символ поиска по тегу
  - `>` — символ иерархического поиска путей.
  - `^` — символ для поиска с условным переключателем (конкретные типы и значения приведены ниже).- `^c` — отображает только документы с галочкой «Является ли категория»
    - `^d` — отображает только документы с галочкой `Is Dead/Gone/Destroyed`.
    - `^f` — отображает только документы с галочкой «Завершено».
    - `^m` — отображает только документы с галочкой «Является ли второстепенный документ», которые обычно невидимы и отфильтровываются.

## Фильтрация полного поиска

Эта функция предназначена в основном для тех, кому нужен полномасштабный поиск, который может сканировать любое поле в любом документе для сопоставления значений практически в любом месте данных. Фильтрация полного поиска позволяет сузить результаты, выполняя поиск по всей базе данных документов и точно определяя, что вам нужно.

### Несколько слов предостережения

**Полный поиск — очень мощный, но требовательный инструмент: чем больше становится ваш проект, тем более требовательным он становится. Если у вас, например, более 2000 документов и поиску приходится сканировать их все, полный поиск может занять несколько секунд, чтобы обновить результаты. Имейте это в виду: это может включать в себя большой объем данных.**

### Фильтрация работает следующим образом и следует этим правилам.

- **Полный поиск можно использовать в сочетании с любыми другими фильтрами и/или обычными поисковыми запросами**
- **В поиске одновременно может присутствовать только один экземпляр полного поиска**
- **Фильтр не чувствителен к регистру, что означает, что вы можете вводить все в ВЕРХНЕМ или нижнем регистре, это не имеет значения**
- **В случае списков и множественных связей все введенные значения преобразуются в одну длинную текстовую строку для удобства поиска**
  – Пример с полем «Местные валюты»:
    - Исходные значения: «Канадский доллар», «Американский доллар», «Евро», «Клингонский дарсек».
    - Преобразованные значения: `канадский-доллар-американский-доллар-евро-клингон-дарсек`
- **В строке поиска необходимо использовать следующие условия фильтра**
  - `%` - Символ начала полного поиска
  - `:` - Символ разделителя между именем поля и значением поля.
- **Можно использовать точный поиск**
  - И имя поля, и его значение могут быть заключены в отдельные разделители.
  - Пример для обоих точных: `%"local-currency":"some-currency"`
  - Пример точного имени поля: `%"local-currency":some-currency`- Пример точного значения поля: `%local-currency:"some-currency"`

- **Если термин фильтра содержит пробелы, замените их символом `-`**
  - Пример: вы хотите найти поле под названием «Местные валюты», которое содержит в качестве значения «Канадские доллары»; чтобы полностью соответствовать этому полю и значению, введите `%local-currency:canadian-dollars`
- **Можно выполнить полнотекстовый поиск, проверив все поля на наличие нужного текста, выполнив следующее: `%:canadian-dollars`**
- **Список полей/типов полей, с которыми не работает полный поиск:**
  - Тип поля «Разрыв» (это большие заголовки, присутствующие по всему документу).
  - Тип поля «Теги» (оно покрыто более сложным фильтром тегов)
  - Тип поля `Switch` (это поле не содержит текстовых значений для даже фильтрации и частично покрыто опцией поиска переключателя)
  - Поле `Имя` (это основная задача поиска, и обычный поиск гораздо более продвинут для поиска по нему)
  - Поле «Принадлежит к» (оно охватывает гораздо более продвинутый иерархический поиск пути)
