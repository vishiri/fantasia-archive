# 变更日志
----------

## 2.4.14 - 版本提升

### 新功能
- **本地化**：操作管理器保存失败、项目打开/创建错误回退以及**导入/导出应用程序配置**错误 toast 现在使用每种受支持语言的区域设置文件。 **Fantasia 存档设置** **记录完整活动负载** 帮助文本在所有区域设置中都匹配英语。
- **项目设置**：**保存而不关闭**与**保存设置**同样保存，但保持对话框打开以便继续编辑。

### 错误修复与优化
- **项目设置**：**世界**详情行中的**删除世界**与**世界名称**和**世界颜色**字段垂直对齐。
- **语言选择器**：**Svenska** 现在显示瑞典国旗，而非萨尔瓦多国旗。

## 2.4.13 - 版本提升

### 新功能
- **界面语言**：语言选择器列出**16**界面语言（**英语、美国**、**德语**、**法语**、**意大利语**、**瑞典语**、**芬兰语**、**挪威语**、**葡萄牙语**、**西班牙语**、**Ελληνικά**、**Українська**、**土耳其语**、 **中文**、**日本语**、**हिन्दी** 和 **Русский**）。菜单标签使用本地化的 **AI** 后缀标记 AI 辅助区域设置（字母 **AI** 在每个标签中均保留拉丁文）。
- **翻译**：每种选择器语言都附带菜单、对话框、浮动窗口、通知和应用内帮助降价的完整 UI 字符串（**高级搜索指南**、**高级搜索备忘单**、**提示、技巧和琐事**以及 **变更日志**），从更正后的 **英语、美国** 目录生成。
- **阿拉伯布局**：选择**阿拉伯语**会在整个应用程序中应用从右到左的布局；语言菜单保持从左到右的每行阅读方向，因此本地名可以自然显示。- **桌面安装程序** 在 **Windows**、**macOS** 和 **Linux** 软件包上为 **Open with** 注册 **`.faproject`** 类型；双击项目文件或在系统选择器中选择 **Fantasia Archive** 打开 **`.faproject`** 作为活动会话，并在打开成功时更新 **加载最近的项目**（开发 **`yarn quasar:dev: Electron`** 会跳过自动打开，除非您设置 **`FA_DEV_OPEN_FAPROJECT=1`**）。
- **欢迎屏幕**和**项目管理**：启动欢迎操作和匹配的**项目管理**行使用**创建新项目**和**加载现有项目**（**en-US**、**Deutsch**、**Français**）。- **欢迎屏幕**：当 **项目管理** 具有已保存的 **加载最近的项目** 列表时，**Splash** 显示分割恢复控件：当没有 **`.faproject`** 处于活动状态时，主段显示 **恢复最新项目**（打开最新的最近文件），或当会话已加载时，显示 **恢复当前项目**（返回到该文件而不出现已处于活动状态的警告）。该箭头列出了与菜单相同的最近项目行（名称加路径），并显示 **浏览最新项目** 作为其可访问名称，并在其右侧显示 **Quasar** 工具提示。如果没有保存最近的记录，该控件将保持隐藏状态。
- **应用程序设置**：**隐藏“浏览最新项目”工具提示**删除欢迎屏幕上**恢复最新项目**旁边的插入符号工具提示；当您保存切换而不重新启动应用程序时，启动画面会更新。- **自定义项目 CSS**：对于活动的 **`.faproject`**，**设置和工具** 在 **自定义应用程序 CSS** 旁边列出 **自定义项目 CSS**。可移动的 Monaco 窗口将每个项目的样式表文本保留到 SQLite 中，并具有去抖自动保存功能，在应用程序范围的 **自定义应用程序 CSS** 之后通过 **`#faProjectUserCss`** 应用实时预览，保持 **自定义应用程序 CSS** 和草稿记事板之间的浮动框架，并在加载、切换或导入项目时刷新。每个桌面平台上的默认全局快捷键都是 **Ctrl+Shift+J** （与 macOS 上的文字 Control 键相同），连接为仅打开：它永远不会关闭窗口，并且在编辑器保持打开状态时重复按下不会影响内存中的草稿。- **项目记事板**：**项目管理**增益**切换项目记事板**，在加载**`.faproject`**之前禁用，每个桌面平台上的默认快捷方式**Ctrl+Alt+D**； **关闭**在**应用记事板**等标签下显示静音括号快捷方式。文本和可移动窗口几何形状保留在 SQLite **`project_data`** KV 行中的每个项目中；当两者都打开时，渲染器会将该浮动框保留在 **App Noteboard** 上方，同时两者都保持在模态镶边 (**6000+**) 下方。

- **英文副本**：**应用程序设置**选项标题和描述、导入/导出指南、按键绑定和操作监视器标签以及帮助降价均以**美国英语**校对；暂存 **TODO** 前缀已从设置标签中删除。
- **欢迎屏幕**：**`/`** 和 **`/home`** 现在共享一个 **MainLayout** shell（顶栏菜单、语言选择器和窗口控件保持安装状态）。只有中心页面在欢迎屏幕和工作区之间交叉淡入淡出；左侧抽屉以相同的时间滑入**`/home`**。当 **`.faproject`** 会话变为活动状态时（创建、加载、恢复、最近项目选择或从操作系统打开项目），应用程序会从任何非工作空间路径（欢迎、**错误/未找到**或未来的 **MainLayout** 页面）导航到 **`/home`**，并进行短暂的淡入淡出过渡，**组件测试**线束路径除外。
- **欢迎屏幕**：左侧**基本链接**抽屉完全远离欢迎路线 - **`/`** 上没有预留导轨和抽屉镀铬 - 并在工作区打开时滑入 **`/home`**（与页面淡入淡出的时间相同），而不是以全宽弹出。- **错误/未找到**：未知路线现在使用相同的 **MainLayout** 顶栏作为欢迎和工作区屏幕（**项目管理**、**文档和内容**、**设置和工具**、**帮助和信息**，以及语言选择器和窗口控件），而没有左侧抽屉。
- **错误/未找到**：当**`.faproject`**会话已处于活动状态时，页面会在**返回应用程序开始屏幕**上方显示**恢复当前项目**；它以与欢迎**恢复当前项目**主要部分相同的方式重新打开活动文件，没有最近项目下拉列表或已经活动的警告 toast。- **项目加载**：重新打开已处于活动状态的项目 - 从 **加载现有项目**、**加载最近的项目**、操作系统双击或匹配菜单或启动画面 **加载现有项目**行 - 以与成功加载相同的 **项目名称** 行格式显示 **警告** toast，不重新加载数据库，并在您位于欢迎屏幕或 **错误/未找到** 页面时导航到工作区。 **在**错误/未找到**时恢复当前项目**和欢迎恢复主要部分（**当会话处于活动状态时**恢复当前项目**，**当没有会话时**恢复最新项目**）安静地返回工作区，没有任何警告。
- **欢迎屏幕**：**恢复最新项目**最近项目菜单使用相同的石板页面背景、金色边框、亮金色行标题、静音路径、金色行分隔线、浅色主亮悬停色调以及与顶栏菜单相同的阴影，而不是默认的深青色应用菜单面板。
- **欢迎屏幕**：拆分**恢复最新项目**下拉箭头再次显示其 V 形；继承的大写文本样式破坏了该段内的材质图标连字。- **欢迎屏幕**：欢迎路线上的 **创建新项目** 通过与工作区路线相同的 **AppControlMenus** 对话框主机打开 **新项目**。
- **应用程序设置**：**隐藏欢迎屏幕社交链接**隐藏欢迎屏幕上的金色分隔线和**社交联系**按钮行；当**应用程序设置**中保存的设置发生更改时，启动画面会做出反应。
- **应用程序设置**：**隐藏开始屏幕上弹出的提示**抑制启动**您知道吗？**欢迎路线上的提示通知（在应用程序启动时阅读一次，而不是在每次设置更改时阅读）。
- **应用程序设置**： **跳过欢迎屏幕** 在启动时打开最新的 **`.faproject`** - 或已加载的活动会话文件 - 并导航到工作区（**`/home`**，与左侧抽屉共享 **MainLayout**），而不是停留在欢迎屏幕上；当您在显示欢迎路线时打开设置时，启动画面也会重定向。如果最新的最近文件不再存在，应用程序会显示错误并停留在欢迎屏幕上，而不是打开下一个最近的项目（包括启动后已尝试跳过一次）；由于没有最近的项目，欢迎屏幕仍然出现。- **项目管理**：从菜单中删除了**保存项目**和**关闭项目**，直到这些操作具有真正的桌面实现（**en-US**、**Deutsch**、**Français** 副本已更新）。
- **项目 SQLite**：Main 在数据库句柄旁边保留活动的 **`.faproject`** 路径，以便短暂的不同步（例如在窗口空闲之后）可以重新打开同一文件，重试操作一次，并可选择在 main 丢失这两个文件时向渲染器询问路径。完整渲染器重新加载仍然会清除真实文档导航上的会话，但同一文档和子帧搅动不再单独丢弃句柄。
- **OS-open 握手**：启动启动 **`faProjectOsOpen`** 现在会在 **渲染器就绪** IPC 握手之前短暂等待 **`faContentBridgeAPIs.faProjectOsOpen`**，因此在 Electron **argv** 上使用 **`.faproject`** 路径启动（安装程序关联或 **FA_E2E_OS_OPEN** 端到端重播）在预加载时不会错过连线**`contextBridge`** 表面跟随第一个启动滴答声。- **自定义项目 CSS**：当没有 **`.faproject`** 会话处于活动状态时（例如在欢迎屏幕上或应用程序清除数据库句柄后），从浮动编辑器保存现在显示清晰的 **自定义项目 CSS** 消息，而不是将跳过的 SQLite 写入视为成功，然后显示不匹配错误。
- **项目记事板**：在窗口保持打开状态时加载另一个**`.faproject`**现在会移动框架并调整其大小以匹配该项目保存的几何图形，而不是在屏幕上留下先前项目的足迹。
- **项目记事板**：在去抖文本写入完成之前，仅帧保存往返读取 SQLite 时，移动或调整窗口大小不再清除屏幕上的草稿文本；当合并省略**文本**的补丁的回读时，渲染器保留实时文本区域值。
- **项目记事板**：当 **main** 在渲染器重新加载期间清除了 SQLite 句柄（例如 **Control+F5** 或语言拼写检查刷新）时，保存注释文本或几何图形不再触发错误 toast：延迟的写入会安静地下降，而不是意味着保存往返失败。- **Storybook**：嵌套工作区**内容桥**现在模拟存根**项目记事板**加上**项目样式** KV 读写，因此**WindowProjectNoteboard** 和 **WindowProjectStyling** **Autodocs** 加上画布预览在这些浮动窗口自动保存几何图形、注释文本或 CSS 草稿时不再显示虚假的**操作管理器** 失败。 **mocks** 范围的 **TypeScript** 配置为维护者恢复 **`app/`** 路径类型，而无需将类型检查扩大到整个预览包。
- **Storybook**：**AppControlMenus** Autodocs 组件描述列出当前 **项目管理** 行，以便在菜单更改后与 **`_data/project`** 匹配。
- **测试** — 端到端 **`checkProjectStyling`** 重命名 **`.faproject`** 夹具基线和套件标题，以便 **自定义项目 CSS** 流程描述 Monaco 实时预览、SQLite 持久性、清除样式表和每个项目隔离，而不是不透明的 **alpha** / **beta** 标签。组件线束 **`WindowProjectStyling.playwright.test`** 涵盖浮动镶边、冷加载 **Monaco**、帮助面板主题列表以及 **关闭而不保存** 将存储的项目 CSS 留空。- **测试** — **`checkProjectManagementFlow`** 端到端现在断言启动**恢复最新项目**（**MRU 下拉标签**匹配**项目 → 加载最近的项目**，在拼写检查会话重置后通过拆分主段重新加载，从另一个最近的行加载）。

### 错误修复和优化
- **Playwright**（组件、**E2E** 和 **Storybook** 视觉）使用终端 **`list`** 报告器：每个测试在运行过程中打印在自己的行上（HTML 报告不变）。

- **自定义应用程序 CSS** 浮动窗口 - 标题文本不再强制 **`margin: 0`**，因此标题在拖动行和帮助图标旁边保持正常间距。
